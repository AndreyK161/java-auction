# Auction Backend Service (MVP)

Профессиональный бэкенд-сервис для платформы интернет-аукционов, построенный на микросервисной архитектуре с использованием фреймворка Spring Boot. Проект реализует асинхронную обработку ставок, кэширование высоконагруженных запросов, надежное реляционное хранение данных и централизованную систему аутентификации и авторизации (IAM).

---

## Стек технологий

* **Backend:** Java 17, Spring Boot (Data JPA, Web, Security)
* **Безопасность (IAM):** Keycloak (OAuth2 / OpenID Connect)
* **Базы данных:** PostgreSQL (основная БД), Redis (кэширование)
* **Асинхронное взаимодействие:** RabbitMQ (брокер сообщений)
* **Документирование API:** Swagger UI / OpenAPI 3.0
* **Контейнеризация:** Docker, Docker Compose

---

## Архитектура и путь запроса (Data Flow)

Приложение спроектировано с учетом высоких требований к безопасности и производительности при пиковых нагрузках (интенсивные торги в последние минуты аукциона).

1. **Аутентификация:** Пользователь проходит авторизацию через **Keycloak**, получая JWT-токен доступа.
2. **Запрос к API:** Запрос с токеном поступает на слой контроллеров, задокументированных в **Swagger UI**. **Spring Security** валидирует токен в Keycloak, проверяя права доступа пользователя (Roles/Scopes).
3. **Асинхронные события:** При отправке новой ставки запрос мгновенно уходит в очередь **RabbitMQ**, что изолирует бэкенд от пиковых нагрузок и гарантирует обработку каждой ставки.
4. **Бизнес-логика:** Сервис **Spring Boot** вычитывает сообщения из очереди и обрабатывает их.
5. **Кэширование:** Данные о популярных лотах и активных торгах кэшируются в **Redis** (с настроенным TTL), снижая нагрузку на основную БД при частых запросах страниц.
6. **Персистентность:** Финальное состояние лотов, пользователей и транзакций сохраняется в реляционную базу **PostgreSQL**.

## Быстрый старт и развертывание

Для удобства локальной разработки и тестирования все инфраструктурные сервисы контейнеризированы с помощью Docker Compose.

### Предварительные требования
* Установленный [Docker](https://www.docker.com/) и Docker Compose
* JDK 17 (для сборки самого приложения из исходников)

### 1. Клонирование репозитория
```bash
git clone [https://github.com/your-username/auction-backend.git](https://github.com/your-username/auction-backend.git)
cd auction-backend

```

### 2. Настройка окружения

Создайте файл `.env` в корневом каталоге проекта на основе ваших параметров и укажите путь для хранения данных контейнеров:

```env
POSTGRES_VOLUME_PATH=/your/local/path/postgres_data

```

### 3. Запуск инфраструктуры в Docker

Запустите базу данных, брокер сообщений, кэш и сервер авторизации одной командой:

```bash
docker-compose up -d

```

После этого локально развернутся и будут доступны следующие сервисы:

* **Keycloak (IAM):** `localhost:8080` (Панель администратора, управление пользователями и ролями)
* **PostgreSQL:** `localhost:5432` (БД: `auction`, логин/пароль: `admin/admin`)
* **RabbitMQ:** `localhost:5672` (Панель управления: `http://localhost:15672`, логин/пароль: `guest/guest`)
* **Redis:** `localhost:6379`

### 4. Сборка и запуск Spring Boot приложения

Соберите проект с помощью Maven и запустите его (приложение настроено на порт `8081`, чтобы не конфликтовать с Keycloak):

```bash
./mvnw clean package
java -jar target/auction-service-0.0.1-SNAPSHOT.jar

```

## Документация API

После успешного запуска приложения полная интерактивная спецификация всех эндпоинтов (REST API) доступна по адресу:

```text
http://localhost:8081/swagger-ui.html

```

Через интерфейс Swagger UI можно выполнять тестовые запросы, отправляя полученный от Keycloak токен Bearer для проверки защищенных методов.

---

## Безопасность и валидация

* Полноценная ролевая модель авторизации (RBAC) на базе Keycloak и Spring Security.
* Входные данные запросов строго валидируются на уровне контроллеров Spring (JSR-380).
* Взаимодействие между компонентами системы изолировано внутри внутренней конфигурации докер-сети.
