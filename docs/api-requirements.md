### Правила составления спецификации

#### Путь определения сервиса

`/api/v1/<имя сущности во множественном числе>/`

#### Методы CRUD

- Create -> POST. Пример: `/api/v1/lots/ + body`. Ответом является созданная сущность (ДТО представление)
- Get -> GET. Пример: `/api/v1/lots/{id}`. Ответом является сущность (ДТО представление)
- List -> GET. Пример: `/api/v1/lots/?page=1&count=10` (ДТО представление элемента списка может быть отличаться от детального элемента)
- Update -> PATCH. Пример `/api/v1/lots/{id} + body`. Ответом является обновленная сущность (ДТО представление)
- Delete -> DELETE. Пример `/api/v1/lots/{id}`. Ответом является код 200, означающий успешное удаление сущности (удаление происходит виртуально)

#### Бизнес методы

Всегда лучше разрабатывать АПИ с использованием бизнес методов.

Использовать POST метод REST с суффиксом действия в конце uri пути.

Пример:
`POST /api/v1/lots/{id}/move-to-trash/ + (empty body)`
