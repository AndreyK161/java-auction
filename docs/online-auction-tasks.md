### Задачи реализации онлайн аукциона

#### Обработка реестра лотов (Вступительная часть)

1. Реализация предметной области сущностей лотов с полями и связями (лот, изображение, продавец)
2. Подключение логирования
3. Реализация хранения изображений лотов (желательно воспользоваться уже реализованным сервисом)
4. Реализация регистрации лота через REST (с валидацией)
5. Реализация изменения информации по лоту (конкретному) через REST (с валидацией)
6. Получения списка лотов по фильтру с постраничной навигацией
7. Удаление лота (без реального удаления из БД)

#### Проведение торгов (Основная часть)

1. Проектирование предметной области (ERP диаграмма)
2. Реализация предметной области (миграции данных, сущности, интерфейсы сервисов)
3. Подключение почтового сервиса < !!!!надо понять какого, подсмотреть у Паши
4. Регистрация пользователя через почту
5. Аутентификация пользователя по логину и паролю, с дальнейшей работой через GWT токен (токен, +refresh токен)
6. Восстановление пароля пользователя через почту
7. Получение списка пользователей-участников системы
8. Публикация предстоящих торгов с регистрацией участников и лотов
9. Редактирование информации по опубликованным торгам
10. Регистрация/(добавление администратором) участника в опубликованные, активные торги
11. Добавление/удаление лота в опубликованные торги администратором
12. Получение списка торгов с фильтром по статусу, периоду проведения, постраничной навигацией
13. Получение информации по торгам со списком участников и лотов, доступна аутентифицированному пользователю
14. Администратор торгов пополняет счет участникам
15. Начало торгов из списка опубликованных
16. Начало торгов по лоту
17. Реализация ставок по лоту
18. Завершение торгов по лоту (через 1 минуту с момента последней ставки) с определением победителя
19. Завершение торгов
20. Списание с ЛС победителя средств за покупку лота
21. Пополнение ЛС продавца проданного лота
22. Получение отчета по завершенным торгам
23. Отправка отчета об итогах торгов участникам (через 1 час после завершения, должно это настраиваться)
24. Получение детальной информации по участнику приобретенных лотах, участия в торгах
25. Получение информации по счету участника с историей движения средств (пополнение счета администратором, продажа лота такого-то, покупка лота такого-то)

#### Онлайн табло

1. Для активных торгов должно быть доступно табло, которое отображает лоты, участвующие в данный момент в торгах,
с информацией о: данные лота, начало торгов по лоту, количество ставок, текущая принятая ставка, начальная цена.
2. Реализация демонстрации нагрузки на табло. Приложение нужно уронить
3. Кеширование информации с выдержкой нагрузки из предыдущего задания. Задержка данных не более 10 сек. Табло должно справляться с высокими нагрузками - 1M rps

#### Статистика и отчетность

1. История приобретенных лотов участником торгов
2. Формирования отчета по торгам сразу же после окончания с историей ставок в виде документа

#### Нагрузочное тестирование

1. После разработки приложения необходимо провести его нагрузочное тестирование:
   - Прежде всего необходимо указать адрес, по которому docker сможет подключиться к хостовой машине. 
   - По умолчанию указан адрес `host.docker.internal`. Этот вариант подходит для Windows и Mac. 
   - Для Linux необходимо заменить его на `172.17.0.1` - это адрес, используемый по умолчанию. 
   Если он не подходит, то необходимо выполнить команду в терминале `ip addr show docker0` и посмотреть актуальный адрес.
   Данный адрес необходимо прописать в [stress-test.jmx](../stress-test.jmx) (секция `HTTPSampler.path`) 
   и в [.env](../env) (переменная `TARGET_HOST`)
   - После всех приготовлений необходимо запустить тест, для этого в [docker-compose.yaml](../docker-compose.yaml) 
   необходимо запустить `jmeter`. Он сам подхватит файл с описанием теста и начнет его выполнять. В терминале контейнера
   можно отслеживать ход выполнения теста. По окончании минуты будет выведен результат формата 
   `summary =   1000 in 00:01:00 =   16.7/s Avg:     6 Min:     3 Max:   216 Err:     0 (0.00%)`. Необходимо убедиться,
   что количество ошибок (Err) равно нулю.
   - При возникновении ошибок их причины можно найти в файле `jmeter_results\result.jtl`
